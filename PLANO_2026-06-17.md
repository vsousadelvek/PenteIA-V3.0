## Gaps pendentes — prioridade do dia

### 🔴 GAP 5 — KEV trigger automático de re-simulação
**Problema:** Scheduler existe (`APScheduler` em `app_fastapi_v2.py`) mas só roda em horário fixo. Não reage a eventos externos.

**Solução:**
1. `epss_engine.py` já baixa KEV real com cache de 12h
2. Comparar KEV atual vs KEV anterior (salvo em DB ou arquivo)
3. Se nova CVE entrou no KEV E está presente nos assets do cliente → dispara BAS automaticamente
4. Notifica via WebSocket (já implementado)

```python
# Lógica a adicionar no scheduler job
prev_kev = load_previous_kev()
curr_kev = epss_engine.get_kev_list()
new_entries = set(curr_kev) - set(prev_kev)
if new_entries:
    for client in get_active_clients():
        if new_entries.intersection(client.tracked_cves):
            trigger_bas_simulation(client.target, playbook="web_app_full_audit")
            notify_websocket(client.user_id, f"Nova KEV detectada: {new_entries} — BAS disparado")
save_current_kev(curr_kev)
```
