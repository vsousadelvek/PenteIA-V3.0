## Gaps para ser market-leader (priorizados)

### 🔴 GAP 1 — Execução real de técnicas no endpoint (CRÍTICO)
**Problema:** `_simulate_execution()` sempre retorna True. Zero execução real.  
**Impacto:** Sem isso, não é BAS — é scanner web com labels MITRE.  
**Solução:** Implementar no `execution_engine.py`:
- T1105: download de arquivo de URL controlada PenteIA
- T1070: write + delete de arquivo de teste em %TEMP%
- T1547: write temporário em Run key + rollback automático
- T1059: executa `whoami` via subprocess e captura output real
- T1003 safe: dump de processo não-sensível (notepad) em sandbox

### 🔴 GAP 2 — Integrar Anthropic API no ai_module.py (rápido, alto impacto)
**Problema:** AI em demo mode — keyword matching com strings hardcoded.  
**Solução:** Substituir fallback por chamada `claude-haiku-4-5-20251001`:
```python
import anthropic
client = anthropic.Anthropic(api_key=os.getenv("ANTHROPIC_API_KEY"))
msg = client.messages.create(model="claude-haiku-4-5-20251001", max_tokens=1024, ...)
```
Custo: centavos por análise. Output: narrativa real de pentest.

### 🔴 GAP 3 — LDAP enumeration real no AD engine
**Problema:** `ad_attack_engine.py` retorna `"status": "simulated"` em tudo.  
**Solução mínima (read-only, zero risco):**
```python
import ldap3
conn = ldap3.Connection(server, user=user, password=pwd, authentication=ldap3.NTLM)
conn.search('dc=corp,dc=local', '(servicePrincipalName=*)', attributes=['sAMAccountName','servicePrincipalName'])
# detecta SPNs Kerberoastable sem executar o ataque
```

### 🟡 GAP 4 — SIEM correlation robusta
**Problema:** Matching remove `.` de technique IDs — `T1003.006` → `T1003006`, falha.  
**Solução:** Mapa canônico technique_id → aliases + queries por vendor:
- Wazuh: query `ossec.log` por `rule.mitre.id`
- Elastic: query `threat.technique.id`
- Splunk: SPL `index=* mitre_technique_id=...`

### 🟡 GAP 5 — KEV trigger automático de re-simulação
**Problema:** Scheduler existe mas não é inteligente.  
**Solução:** `epss_engine.py` já baixa KEV real. Adicionar:
```python
# Se CVE presente no ambiente entrar no KEV → dispara BAS automaticamente
new_kev = set(current_kev) - set(previous_kev)
if new_kev.intersection(client_cves):
    trigger_bas_simulation(playbook="web_app_full_audit", target=client_target)
```

### 🟡 GAP 6 — Safe payload para EDR validation
**Problema:** Sem prova real que EDR detectou o ataque.  
**Solução:** Script Python assinado que:
1. Abre conexão TCP para `pentest-c2.penteia.com.br:443`
2. Escreve `penteia_test_YYYYMMDD.tmp` em %TEMP%
3. Auto-deleta após 30s
O cliente verifica se o EDR alertou. Isso fecha o loop real.

### 🟢 GAP 7 — Benchmarking entre clientes (médio prazo)
Banco de dados anônimo opt-in com percentis por setor (saúde, financeiro, varejo).
"Seu score de 41% está abaixo da média de 58% do setor financeiro BR."
